UD Pattern Library for Game and Interactive Media Designers

Accessibility is Not an Afterthought.

The modern digital world, interactive games and media to websites and browsers, often has barriers for a significant amount of the population. This exclusion and barrier is not a technical necessity; it is a flaw in the design process itself (Heron, 2012).

The Problem: Oversight, Not Impossibility

Universal Design (UD) is defined as "the design of products and environments to be usable by all people, to the greatest extent possible, without the need for adaptation or specialized design" (Mace, 1997). UD is now the ethical and practical framework for digital creation.

The core thesis of this project is that inaccessibility in digital media is most commonly due to oversight or lack of awareness. Designers usually overlook the diverse needs of a population. When games are not accessible, it is often through the omission of common features (Heron, 2012) This oversight is a huge issue. The inaccessibility of games leads to a large number of potential customers being lost.

The Social Model of Disability

This project aligns with the Social Model of Disability, which asserts that disability is caused by the way society is organized, rather than by a personâ€™s impairment or difference. Barriers are created by a lack of access, poor design, and social exclusion, not by the individual (Heron, 2012; People With Disability Australia, n.d.).

The Solution

By incorporating universal design, game and website developers can shift the responsibility from the individual with the disability to the design of the product (Ng, 2017; Burgstahler, 2009).

Universal Design should be incorporated into the regular content lifecycle and not just treated as a task you can check off on a list (Ng, 2017).

The Curb Cut Effect

This approach leverages the Curb Cut Effect: features originally designed for users with disabilities (the "extraordinary users") often improve the overall user experience for all users (Heron, 2012; The Oxford Review, 2023). For example, good color contrast increases visibility for all users in poor lighting conditions or on low-brightness screens (Ng, 2017).

Improved User Experience

When features are designed and created for users with disabilities (the "extraordinary users"), the overall user experience is improved for all users (Heron, 2012; Ng, 2017). For example, good color contrast increases visibility for all users in poor lighting conditions or on low-brightness screens (Ng, 2017).

This design pattern library aims to provide the actionable knowledge that designers need to be able to overcome the barrier of not knowing how to implement Universal Design (Chanchiewvichai et al., 2024).

Pattern Library: The "How" of Accessibility

Practical, theory-backed solutions to integrate Universal Design principles into your interactive media projects.

1. Customizable Input & Pacing

(UD Principle 2: Flexibility in Use)

This pattern supports a wide range of individual preferences and physical abilities by allowing users to have a choice in methods of use and adaptability of the game pace.(Stephanidis et al., 2001) Always give users an option to play games at different speeds and allow them to fully customize the controls to reduce the need for fast reflexes or precision (Heron, 2012).

Implementation Steps (Controls):

  • Full Control Mapping: Allow for full button remapping so users can map the most useful inputs to their most appropriate physical controls (Heron, 2012).
  • Input Toggle: Use click-to-toggle mechanics for actions that would typically require the user to hold buttons down, which is a feature liked by players with mobility or dexterity impairments (Heron, 2012).
  • Pacing Controls: Offer the option to play the game at different speeds like half speed or quarter speed to help users with control issues (Heron, 2012).

2. Semantic Menu Hierarchy

(UD Principle 3: Simple and Intuitive Use)

This pattern ensures the design is easy to understand, minimizing unnecessary complexity and being consistent with user expectations (Stephanidis et al., 2001). Utilize clear and consistent structure of content and use plain language to make sure all information and content is quickly understood and navigable.

Implementation Steps (Web/Menu):

  • Headings for Navigation: Use HTML headings in a logical outline to structure the text to allow users of assistive technology to be able to skip content by heading.
  • Clear Language: Make sure content is written in a way that is clear and concise, and use simple wording and avoid jargon. (Ng, 2017).
  • Descriptive Links: Ensure link title/text clearly relays the purpose of the link, avoiding uninformative phrases like "read more" or "info" (Ng, 2017).

3. Multimodal Status & Feedback

(UD Principle 4: Perceptible Information)

This pattern ensures that essential information is communicated effectively regardless of a user's sensory abilities (Ng, 2017). You should never rely on a single sensory channel to convey information to the user. For example, do not use only color or only sound to communicate the message. Critical information must be presented using different modes (pictorial, verbal, tactile) for redundant presentation (Stephanidis et al., 2001).

Implementation Steps (HUD/GUI):

  • Color-Independent Cues: Utilize patterns, shapes, or symbols alongside color to differentiate elements like teammates and enemies, as relying on color alone is a disadvantage for color-blind users (Heron, 2012; Ng, 2017).
  • Textual Alternatives: Provide subtitles and closed captions (Ng, 2017) for all dialogue and essential sound information. For example if there is audio of a gunshot played, have a small text snippet like: [gun fired close by] (Heron, 2012).
  • Text Customization: Allow players to adjust subtitle and text sizes and colors to support users with varying degrees of eyesight impairment (Heron, 2012).

4. Failsafe & Error Recovery

(UD Principle 5: Tolerance for Error)

This pattern reduces the consequences of accidental or unintended actions as it ensures the design provides fail safe features (Stephanidis et al., 2001).

  • Description: Designs must prevent hazards and allow users to be able to easily recover from mistakes they may make (Stephanidis et al., 2001).
  • Implementation Steps (Gameplay/Menu):
  • Progress Gates: Include a feature in the game or design that allows the player to "skip" a mandatory, skill-intensive section after they failed the section multiple times. This will ensure they are not prevented from completing the game (Heron, 2012).
  • Avoid Autoplay: Disable autoplay for all embedded media, as can be distracting and can cause the user's cursor or page focus to have issues which can interrupt screen readers (Ng, 2017).
  • User Control & Freedom:Include controls for media elements to play and pause at a minimum, and ensure they have keyboard functionality (Ng, 2017).

5. Scalable, Spaced Controls

(UD Principle 7: Size and Space for Approach and Use)

This pattern will ensure appropriate size and space for approach, reach, use, and manipulation despite the size of the user's body or ability of mobility (Stephanidis et al., 2001).

  • Description: Interactive elements must be well spaced, large and easy to reach or select with little effort (Stephanidis et al., 2001).
  • Implementation Steps (GUI/Touch Screens):
  • Reachability: Make reach to all components comfortable by ensuring controls are large enough and spaced apart to prevent clicking buttons/keys by mistake (Stephanidis et al., 2001).
  • Clear View: Provide a clear line of sight to essential elements for both seated and standing users (Stephanidis et al., 2001) by avoiding busy and overlapping interface elements (Ng, 2017).
  • Text Legibility: When text cannot be customized, offer a zoom in option as an alternative to standard text resizing to ensure legibility (Heron, 2012).
